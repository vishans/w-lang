The meta: clause
The meta: clause allows us to specify meta data.
Yep. That's pretty much it. That's all it's for.
Here's an example below. 
    meta: 
        debug
        save
        strict
Now, most attributes in meta: clause are Booleans.
In workout-language there's an easy way to set a Boolean to True. You just write the attribute as is. For instance, in the example above, just writing debug is equivalent to writing debug=True.
If you want to set an attribute to False however, you need to explictly write the assignment. For instance if I were to set strict to False, I would have to write strict=False. There is no shortcut (yet).
!attribute will come in a future update of the language. To avoid writing attribute=False.
I said most of the attributes are Booleans. One of them is a String. That attribute is the output-dir attribute. To assign a value to output-dir, you could do,
output-dir="C:/User/MyFolder". This is a perfectly legal statement.
But since it's the only String in meta:, you can just write "C:/User/MyFolder".
The interpreter is smart enough to figure out that the String belongs to output-dir. You don't have to explicitly state it.
In meta: clause, you don't need to put each attribute in a new line. I just think it looks cleaner. If you want, you can use a one-liner! Or you can mix it: use one-liners and new lines.
All the examples below are equivalent. 
    meta: 
        debug save strict "C:/User/MyFolder"
        
    meta: 
        save debug 
        strict 
        "C:/User/MyFolder"
        
    meta: 
        save debug
        "C:/User/MyFolder" 
        strict=True
In meta: clause, the order in which the attributes appear is not important. This might not be the case for other clauses; which brings me to the next section: how does the interpreter work? How does it figure out the order of the attributes?
It's not actually complicated and you'll see that the interpreter proceeds in a logical manner.
Before we go though, if you're asking if one needs to specify all the attributes, the answer is yes and no. You see, you can set default values for every attributes. If you've set a default value for a particular attribute, the latter will implicitly be assigned that default value unless of course you re-assign it. But if the attribute has no default value, then you would have to implicitly assign a value to it.
We'll talk more about that in a bit.
If you're curious, the list of all the default attibutes of the meta: clause is here. And yes, you can add your own attributes too if need be.
That's it for the meta: clause. Before taking a look at workout: and set: clauses, we need to take a glance at how the interpreter works.
